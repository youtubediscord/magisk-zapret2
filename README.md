# Zapret2 для Android (Magisk модуль)

Модуль для обхода блокировок YouTube, Discord и других сайтов на Android.

<img width="1916" height="821" alt="image" src="https://github.com/user-attachments/assets/d0852896-8aca-4d1e-ad1b-8f269003c5c6" />

## Что это и зачем нужно?

Когда вы открываете YouTube или Discord, ваш провайдер видит это и блокирует соединение. Этот модуль "маскирует" ваш трафик так, что провайдер не может его распознать.

Интересный факт: Телефон с таким запретом можно использовать как роутер и раздавать сразу разблокированный интернет через hotspot

```
📱 Ваш телефон → 🛡️ Zapret2 (маскировка) → 🌐 Провайдер (не видит) → ✅ YouTube работает!
```

**Это НЕ VPN** — трафик идёт напрямую, без сторонних серверов. Работает быстрее VPN и не требует подписки.

---

## Требования

- Android 7.0 (API 24) или новее на ARM/ARM64; сборок для x86/x86_64 нет.
- Magisk 20.4 или новее и рабочий root-доступ. KernelSU и другие root-менеджеры пока не входят в проверенную матрицу релиза.
- Ядро и firewall должны поддерживать IPv4 `iptables`, target `NFQUEUE` с `--queue-bypass`, matches `multiport` и `mark`. При наличии `connbytes` применяется обычное ограничение первых пакетов; без него модуль использует предусмотренный оригинальным zapret2 исходящий режим `KEEPALIVE` и не создаёт входящие правила. Для обработки IPv6 нужен тот же обязательный набор в `ip6tables`; отсутствие только `connbytes` также включает исходящий режим.
- Приложение Zapret2 Control опционально и нужно только для графического управления.

## Установка за 3 шага

### Шаг 1: Скачайте файлы

📥 Скачайте из **[Releases](https://github.com/youtubediscord/magisk-zapret2/releases)**:
- `zapret2-magisk-v*.zip` — модуль для Magisk
- `zapret2-control-v*.apk` — приложение для управления (опционально)

### Шаг 2: Установите модуль

1. Откройте **Magisk**
2. Перейдите в **Модули** → **Установить из хранилища**
3. Выберите скачанный ZIP файл
4. Нажмите **Перезагрузить**

ZIP предназначен только для установки из работающего Magisk. Прошивка через TWRP или
другой custom recovery не поддерживается: безопасное обновление выполняется штатно через
`/data/adb/modules_update`, не затрагивая активный модуль до перезагрузки.

### Шаг 3: Установите приложение (опционально)

1. Откройте скачанный APK файл
2. Разрешите установку из неизвестных источников
3. Установите приложение **Zapret2 Control**

После перезагрузки модуль запустится автоматически. Откройте YouTube — должен работать!

## Android-приложение

Точная матрица поддержки, объяснение разрешений APK, правила миграции и сохранения
пользовательских данных, восстановления, безопасного удаления и диагностический порядок
собраны в [`docs/USER_OPERATIONS_RU.md`](docs/USER_OPERATIONS_RU.md).

**Zapret2 Control** — приложение на Jetpack Compose с интерфейсом Material 3 Expressive. Адаптивная оболочка подстраивает навигацию и содержимое под компактные телефоны и широкие экраны. Интерфейс поддерживает светлую и тёмную темы, а на совместимых версиях Android — динамические цвета системы. Текущая реализация закреплена на экспериментальном Material 3 `1.5.0-alpha24`; это не стабильный API, а правила обновления и снятия opt-in описаны в [`docs/MD3E_EXPERIMENTAL_AND_TESTING.md`](docs/MD3E_EXPERIMENTAL_AND_TESTING.md).

Полная трассировка экранов, владельцев состояния, привилегированных операций и границы между готовностью исходников и обязательными release/device-проверками находится в [`docs/MD3E_SOURCE_COMPLETION_AUDIT.md`](docs/MD3E_SOURCE_COMPLETION_AUDIT.md).

Основные экраны приложения:

1. **Control («Управление»)** — состояние модуля и сервиса, запуск, остановка, перезапуск, автозапуск, проверка обновлений, полный откат и необратимое чистое удаление всех данных с сохранением установленного APK.
2. **Profiles («Профили»)** — профили активного пресета: `--skip`, имя, hostlist/ipset, готовая стратегия и порядок выполнения.
3. **Presets («Пресеты»)** — выбор готового TXT-пресета, полный текстовый редактор и просмотр точного итогового `nfqws2` argv.
4. **Hostlists («Хост-листы»)** — список хост-листов, безопасный импорт/обновление `.txt`-файлов и переход к их содержимому.
5. **DNS** — параметры DNS, используемые модулем.
6. **Hosts editor («Редактор hosts»)** — просмотр и редактирование записей hosts.
7. **Logs («Логи»)** — просмотр, фильтрация, копирование и отправка диагностического вывода.
8. **About («О приложении»)** — сведения о приложении, модуле и версиях.
9. **Hostlist content/details («Содержимое хост-листа»)** — просмотр и редактирование записей выбранного хост-листа.

Операции, которым нужны root-доступ или ожидание ответа модуля, показывают состояние выполнения и результат прямо в соответствующем экране.

Пресет — полный TXT-набор запуска. Профиль — один блок между `--new`: фильтры,
состояние `--skip` и упорядоченные `--lua-desync`. Готовая стратегия является только
шаблоном, который заменяет строки `--lua-desync` выбранного профиля. Отдельного
постоянного состояния категорий или выбранных стратегий нет.

`runtime.ini` хранит только имя `active_preset` и общие безопасные параметры запуска.
Его проверка и явное восстановление вынесены в отдельный `runtime-config` механизм:
чтение не изменяет файл, единственным валидатором является shell-парсер модуля,
а APK показывает его ограниченный диагностический envelope без enum-копии кодов.
Собственный компилятор проверяет TXT и зависимости, вычисляет точное объединение портов
включённых профилей, создаёт закрытый argv-артефакт и передаёт аргументы `nfqws2`
без `eval`, shell-разбиения и `@config`. TCP и UDP перехватываются только если они
нужны активному пресету; voice-профиль автоматически добавляет UDP 3478, 5349 и 19302.

## Выбор пресета и стратегий

Если YouTube/Discord не работает после установки, попробуйте другую стратегию:

### Рекомендуемые стратегии

| Сервис | Стратегия по умолчанию | Альтернатива |
|--------|----------------------|--------------|
| YouTube | Syndata + Multisplit | Censorliber v2 |
| Discord | Syndata + Multisplit | Censorliber |
| Discord голос | — | Voice UDP (STUN) |
| Telegram | Syndata + Multisplit | Censorliber |
| QUIC/UDP | QUIC Fake x6 | QUIC Fake x11 |

### Как выбрать стратегию профиля:

1. Откройте приложение **Zapret2 Control**
2. Откройте раздел **Профили**
3. Выберите нужный профиль
4. Выберите стратегию из списка
5. Сервис перезапустится автоматически

> 💡 **Совет:** Если одна стратегия не работает — пробуйте следующую. У разных провайдеров разные методы блокировки.

### Раздел «Пресеты»

- Быстрый выбор готовых TXT-пресетов из `zapret2/presets/*.txt`
- Пресет можно отредактировать прямо в приложении и сохранить
- Вкладка «Итоговая команда» собирает текущий, даже несохранённый, текст тем же
  компилятором, показывает точные TCP/UDP-порты и позволяет скопировать argv
- По умолчанию выбран `Default v1 (game filter).txt`
- Windows-only и неизвестные параметры, а также `--ipcache*`, отклоняются до запуска

## Решение проблем

### ❌ YouTube/Discord всё ещё заблокирован

1. Откройте Zapret2 Control → Профили → попробуйте другую стратегию
2. Дождитесь перезапуска сервиса
3. Подождите 10 секунд и проверьте

### ❌ Модуль не запускается

Ваше ядро или firewall могут не поддерживать обязательный набор NFQUEUE. Одна только запись в `/proc` этого не доказывает: запуск требует `NFQUEUE` с `--queue-bypass`, а также `multiport` и `mark`. `connbytes` проверяется отдельно: если kernel match отсутствует, используется исходящий режим `KEEPALIVE` оригинального zapret2. Для безопасной проверки релизного устройства используйте read-only preflight:

```bash
sh tests/device/smoke.sh --serial ТОЧНЫЙ_ADB_SERIAL --evidence-dir /безопасный/каталог/evidence
```

Полный порядок, явные предохранители для изменяющих стадий и проверка SHA-256 точных подписанных артефактов описаны в [`tests/device/README.md`](tests/device/README.md). По умолчанию harness ничего не изменяет на устройстве и сохраняет закрытый before-state bundle. Capability manifest и machine-схемы проверяются в точном порядке без лишних или повторных полей; IPv4 dump обязателен, а IPv6 требует либо успешный dump, либо каноническую запись `not_available`.

Для быстрой ручной диагностики доступны неполные сигналы:
```bash
su -c "[ -f /proc/net/netfilter/nfnetlink_queue ] && echo nfnetlink_queue:ok || echo nfnetlink_queue:missing"
su -c "[ -f /proc/net/netfilter/nf_queue ] && echo nf_queue:ok || echo nf_queue:missing"
su -c "grep -q NFQUEUE /proc/net/ip_tables_targets 2>/dev/null && echo iptables_target:ok || echo iptables_target:missing"
```
На современных Android-ядрах файл `/proc/net/netfilter/nf_queue` может отсутствовать — это нормально.
Эти команды не проверяют `--queue-bypass`, `connbytes`, `multiport` и `mark`, поэтому не заменяют preflight или транзакционный probe при запуске.

### ❌ Конфликт с AdGuard / NetGuard / AFWall+

Эти приложения тоже перехватывают трафик. Решения:
1. Отключите их временно для теста
2. Или добавьте Zapret2 в исключения

### ❌ Интернет стал медленнее

Отключите ненужные профили переключателем `--skip`.

## Команды терминала

Для продвинутых пользователей (Termux):


## Частые вопросы

### Это безопасно?

Да. Модуль только модифицирует сетевые пакеты, не собирает данные и не отправляет их никуда.

### Это легально?

Использование для личных целей не запрещено. Законы различаются в разных странах.

### Чем отличается от VPN?

| | Zapret2 | VPN |
|---|---------|-----|
| Скорость | Полная | Зависит от сервера |
| Анонимность | Нет | Да |
| Стоимость | Бесплатно | Обычно платно |
| Трафик | Напрямую | Через сервер |

### Работает с мобильным интернетом?

Да, работает с Wi-Fi, 4G, 5G — любое подключение.

### Садит ли батарею?

Минимально. Обрабатываются только первые пакеты соединения.
